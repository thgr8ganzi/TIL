# 네트워크

--------------

## 네트워크 기본

### 네트워크

* 개념
  * 컴퓨터와 같은 노드들이 통신기술을 이용하여 그물망처럼 연결된 통신 이용형태
* 장점
  * 많은 정보 공유
* 단점
  * 보안 문제
* 거리기반 네트워크
  * PAN(Personal Area Network)
    * 5m 전후 인접통신
  * LAN(Local Area Network)
    * 근거리 네트워크
    * 사무실과 소규모 공간 고속 통신 회선
  * MAN(Metropolitan Area Network)
    * LAN 과 WAN 의 중간
  * WAN(Wide Area Network)
    * 광대역 네트워크망 LAN 간 연결

### 네트워크 토폴리지

* 계층형
  * 계층적 구조로 한 노드가 여러 하위 노드를 가짐
  * 관리가 쉽고 확장성이 좋음
* 버스형
  * 모든 노드가 하나의 중앙 버스 라인에 연결
* 성형
  * 모든 노드가 중앙 노드에 연결
  * 중앙 노드에 의존적
* 링형
  * 각 노드가 수환 형태로 두개의 인접 노드와 연결
  * 일반적으로 토큰 패싱 방식 사용
  * 한 노드나 연결에 문제가 생기면 전체 네트워크에 영향을 미침
* 망형
  * 노드들이 ㅓㅅ로 광범위 하게 연결
  * 매우 높은 신뢰성과 내결함성
  * 설치 및 유지보수가 어려움
  * 회선수 : n(n-1)/2

### 데이터 전송

* 아날로그/디지털 전송
  * 아날로그
    * 전송 매체를 통해 아날로그 신호 형태로 전달
    * 신호 감쇠가 심하고 오류 발생 확률 높음
  * 디지털전송
    * 전송 매체를 통해 디지털 신호 형태로 잔딜
    * 거리 감쇠는 없으나 장거리 전송을 위해 리피터 사용필요
    * 리피터로 잡음 제거 신호복원 가능
    * 장거리 전송 적합
* 방향에 따른 구분
  * 단방향 통신
    * 일방적 통신
  * 반이중 통신
    * 양방향 데이터 전송 가능, 동시 전송 불가
  * 전이중통신
    * 양방향 데이터를 동시에 송수신
* 직렬전송/병렬전송
  * 직렬전송
    * 한번에 한비트씩 순서대로
    * 데이터 전송 속도 느림
    * 구축 쉬움
  * 병렬전송
    * 문자 단위등 여러 비트를 동시에 전송
    * 데이터 전송 속도 빠름
    * 흐름 제어 필요
* 동기/비동기 전송
  * 동기식 전송 방식
    * 여러 문제를 포함하는 데이터 블록 단위 전송
    * 모뎀이나 다중화기 등 장치로 타이밍 조정 필요
    * 동기 문자나 플래스 사용하여 데이터 블록의 시작 끝 표시
    * SYN, STX, ETX
  * 비동기식 전송 방식
    * 데이터의 시작과 끝을 Start Bit 와 Stop Bit 로 표시
    * 각 문자 사이에 휴지기간이 있어 스톱 비트 지속 전송
    * 저속 전송 사용
  * 동기/비동기 비교
    * 통신속도 고속/저속
    * 회로 복잡도 복잡/단순
    * 비용 고가/저가
    * 동기제어방식 클럭동기/start,stop 비트
    * 전송단위 블록/문자

## 근거리 통신망(LAN)

### LAN

* 개념
  * 여러대의 컴퓨터와 주변 장치가 통신 네트워크를 구성하여 통신하느망
  * 가까운거리
* 특징
  * 제한된 지역내 다양한 장치간 통신
* 구성요소
  * NIC
  * 리피터
  * 허브
  * 브릿지
  * 라우터
  * 게이트웨이
* 전송방식
  * 베이스밴드
    * 디지털 신호를 변조하지 않고 전송
    * 간단한 송수신 장치, 저렴한 비용
    * 1~2km 거리 제한 그이상은 리피터
    * 트위스트 페어 케이블, 동축 케이블
  * 브로드밴드
    * 디지털 데이터를 아날로그 데이터로 변조
    * 주파수 분할 다중화 방식 사용
    * 동시에 여러 정보 전송 가능
* 프로토콜
  * LLC
    * OSI 모델의 데이터링크
    * 흐름제어, 올처리
  * MAC
    * 물리적 전송
    * CSMA/CD

### LAN 표준 802.x 시리즈

* 802.1
  * LAN 의 전체 구성
* 802.2
  * LLC 관한 규약
* 802.3
  * CSMA/CD 방식의 이더넷 규약
* 802.4
  * 토큰 버스 네트워크 규약
* 802.5
  * 토큰 링 네트워크 규약
* 802.11
  * 무선 LAN(Wi-Fi) 규약
* 802.15
  * 무선 개인 영역 네트워크(WPAN) 표준, 블루투스 포함
* CSMA/CD(Carrier Sense Multiple Access with Collision Detection)
  * 다중 유선 충돌 감지 장치
  * IEEE 802.3 이더넷 LAN 에서 사용
  * 다른 사용자의 채널 사용여부 충돌 감지 및 제어
  * 유선네트워크에서 데이터 충돌 감지
  * Manchester 코드 사용하여 전송
  * 네트워크 사용여부 확인 후 전송 또는 대기
  * 충돌 발생시 잼 신호를 보내고 임의 시간 후 재 전송
* CSMA/CA(Carrier Sense Multiple Access with Collision Avoidance)
  * IEEE 802.11 무선 LAN 에서 사용 매체 접근
  * 무선 네트워크의 특성상 충돌 감지가 어려움으로 충돌 회피 기능 포함
  * 네트워크 사용여부 판단후 사용중이면 일정 시간 대기
* 토큰버스
  * 버스형 LAN 에서 사용되는 통신 방식
  * 토근이 논리적으로 형성된 링을 따라 각 노드를 차례로 순회
  * 각 노드가 공평한 송신 권한 가짐
* 토큰 링
  * 링형 LAN 에서 사용되는 통신 방식
  * 물리적으로 연결된 링을 따라 순환하는 토큰을 이용하여 통신
  * 프리토큰, 비지토큰 사용
* 블루투스 규약
  * 802.15.1 : 블루투스 기반 WPAN 규격
  * 802.15.2 : 블루투스 기반 WPAN 규격
  * 802.15.3 : 블루투스 기반 WPAN 규격
  * 802.15.4 : 블루투스 기반 WPAN 규격
  * 802.15.5 : 블루투스 기반 WPAN 규격
  * 802.15.6 : 블루투스 기반 WPAN 규격
  * 802.15.7 : 블루투스 기반 WPAN 규격

### HDLC(High-Level Data Link Control)

* 개념
  * 데이터 링크 계층에서 사용되는 비트지향 프로토콜
  * 신뢰성있는 데이터 전송을 위해 설계
  * 전이중 통신 지원
  * FCS 를 통해 오류 검출
* 프레임구조
  * 플래그
    * 각 프레임의 시작과 끝
  * 주소필드
    * 프레임의 수신자 식별
  * 제어필드
    * 프레임의 유형, 흐름제어 및 오류제어 정보
  * 데이터
    * 실제 전송할 사용자 데이터
  * FCS
    * 프레임의 오류 검출
* HDLC 프레임 유형
  * 정보 프레임 : 데이터 전송
  * 감독 프레임 : 오류 검출, 흐름제어
  * 비순서 프레임 : 추가적 제어 목적
* HDLC 스테이션
  * 주국 : 링크 제어의 책임을 갖고 명령
  * 종국 : 주국 제어하 동작 응답만 가능
  * 혼성국 : 주국 종국의 특징 결함 명령/응답 가능
* 전송 모드
  * 정규 응답 모드(NRM)
    * 하나의 주국 하나 또는 여러 종국
  * 비동기 균형 모드(ABM)
    * 주국이 종국이 될수 있고 종국이 주국이 될수 있음
  * 비동기 응답 모드(ARM)
    * 정규 응답 모드와 비슷하나 주국이 종국이 될수 있음

## 데이터 교환 방식 다중화

### 데이터 교환 방식

* 회성망 종류
  * 전용회선
    * 항상 고정된 통신 회선을 사용
    * 일대일 연결로 송수신
    * 빠른 전송
    * 낮은 오류
    * 고비용
  * 교환회선
    * 교환기를 통해 송/수신 연결
    * 전송 느림
    * 기밀성 무결성 고려
    * 비용 절감
* 교환 방식
  * 회선교환
  * 축척후 교환
    * 패킷교환
      * 데이터그램 방식
      * 가상 회선 방식
    * 메시지 교환
  * 회선교환
    * 교환기 이용 두지점 물리적 접속
    * 연결이 이루어진 전용선처럼 사용
    * 고정된 대역폭
  * 메시지교환
    * 두 노드간의 통신 경로가 미리 설정 안됨
    * 메시지 축척후 전송
    * 비 실시간 전송 적합
  * 데이터그램 방식
    * 논리적 연결 없이 데이터 패킷 독립적으로 전송
    * 각 패킷은 순서와 무관하게 전송
    * 짧은 메시지 전송 효과
  * 가상 회선 방식
    * 회선교환과 데이터 그램 방식 장점 결함
    * 패킷 전송 전 논리적 연결 수행
    * 고정된 경로를 통해 패킷 전송
    * 데이터그램보다 빠르나 많은 동시 사용자는 한계
    * 별도의 호(call) 설정이 있다.

### 다중화

* 개념
  * 하나의 통신 회선을 여러 가입자가 동시에 사용
  * 하나의 통신 회선에 여러 텀널 공유
* 다중화기
  * 여러개 터미널 신호를 하나의 통신 회선을 통해 전송
* 종류
  * 주파수 분할 다중화기(Frequency Division Multiplexer)
    * 물리적 통신 채널을 여러 주파수 채널로 나누어 사용
    * 다양한 주파수로 변조
    * 주파수 분할 완충 지역 Guard Band 주어야함
    * 대역폭 낭비
    * 라디오 TV, CATV
  * 시분할 다중화기(Time Division Multiplexer)
    * 시간을 나누어 여러 사용자가 동시에 사용
    * 디지털 전송 적합
    * 동기식 시분할(Synchronous Time Division Multiplexer)
      * 모든 채널에 타임 슬록이 할당
      * 데이터가 없어도 시간 슬롯 점유
      * 효율성 떨어짐
    * 비동기식 시분할(Asynchronous Time Division Multiplexer)
      * 데이터가 없는 시간은 다른 채널에 할당
      * 효율성 좋음
  * 코드분할 다중화(Code Division Multiplexer)
    * 고유의 코드를 사용하여 여러 데이터 신호 결함
    * 공통 주파수 대역을 통해 여러 사용자 동시에 전송
    * 단일 통신 채널을 여러 사용자가 공유(Code Division Multiple Access)
    * 확산 스펙트럼 기술 사용으로 전송 방해 감지
  * 파장 분할 다중화(Wavelength Division Multiplexer)
    * 광섬유 케이블을 통해 여러 파장의 빛을 전송
  * 공간 분할 다중화(Space Division Multiplexer)
    * 시간이나 주파ㅜ가 아닌 공간 차원에서 다중화
* 역 다중화기와 집중화기
  * 역다중화기
    * 하나의 고속 통신 회선으로부터 데이터를 받아 저속 회선으로 나누어 전송
  * 집중화기
    * 여러 저속 회선으로부터 데이터를 받아 고속 통신 회선으로 합쳐 전송

## 인터넷

### 인터넷

* 개념
  * TCP/IP 프로토콜을 기반 네트워크
  * 1969년 미국 국방부의 ARPANET 에서 시작
  * 쌍방향 통신
  * UNIX 운영체제 관련
* 인터넷 서비스
  * WWW
    * 인터넷상 텍스트, 이미지, 비디오, 오디오등 연결
    * HTTP 프로토콜 하이퍼텍스트 기반
    * 웹브라우저 통해 접근
  * 전자우편
    * SMTP, POP#, MINE 프로토콜 사용
  * 텔넷
    * 원격위치 접속 서비스
  * HTTP
    * 하이퍼텍스트 문서 전송 프로토콜
  * FTP
    * 파일 전송 프로토콜
  * 아키
    * 익명 FTP 정보 검색 서비스
  * 고퍼
    * 정보 검색 서비스
  * 유즈넷
    * 전자게시판 서비스

### IP

* IP 주소
  * 인터넷에서 컴퓨터나 다른 네트워크 장치를 식별할수 있는 고유 번호
  * IPv4
    * 32비트 주소체계
    * 8비트씩 4부분
  * IPv6
    * 128비트 주소체계
    * 16비트씩 8부분
  * 서브넷 마스크
    * 네트워크 부분과 호스트 부분을 구분하기 위해 사용
  * 표시 형식
    * 네트워크 부분 + 호스트 부분
  * 주소분류
    * 유니캐스트
      * 단일 송신자와 단일 수신자 간 통신
    * 멀티캐스트
      * 단일 송신자와 다중 수신자 간 통신
    * 브로드 캐스트
      * 같은 네트워크의 모든 장비에게 보내는 통신
  * IP 주소 클래스
    * A 클래스
      * 0 ~ 127 옥텟
      * 16777216 호스트
      * 128 네트워크
      * 국가/대형 기업
    * B 클래스
      * 128 ~ 191 옥텟
      * 65536 호스트
      * 16384 네트워크
      * 중형 기업
    * C 클래스
      * 192 ~ 223 옥텟
      * 256 호스트
      * 2097152 네트워크
      * 소형 기업
    * D 클래스
      * 224 ~ 239 옥텟
      * 멀티캐스트 주소
    * E 클래스
      * 240 ~ 255 옥텟
      * 연구용 주소
* IPv6
  * 개념
    * IPv4 의 주소 고갈 문제 해결
    * 128비트 주소체계
  * 특징
    * 헤더 처리의 오버헤드 최소화
    * 계층적 주소 할당 체계
    * 자동화된 주소 설정
    * 확장성
  * 표시형식
    * 16비트씩 8부분 128비트로 구성 콜론 으로 구분
  * 헤더
    * 기본헤더(40바이트)
    * 확장헤더
  * 주소분류
    * 유니캐스트
    * 멀티캐스트
    * 애니캐스트
      * 그룹내 가장 가까운 수신자에게 전달
  * IPv4/IPv6 전환기술
    * 듀얼스택
      * 장비들이 IPv4 및 IPv6 모두 지원
      * 두 프로토콜 동시에 처리 가능
    * 터널링
      * IPv6 패킷을 IPv4 패킷에 실어 전송
    * 주소변환
      * IPv4 주소를 IPv6 주소로 변환(헤더변환)

### 서브넷

* 서브넷, 서브넷 마스크
  * 서브넷
    * 하나의 네트워크를 여러개의 작은 네트워크로 분할
    * 라우터를 통해 서로 다른 서브넷 간 통신
  * 서브넷팅
    * 네트워크의 성능 보장 하나의 IP 를 작은 네트워크로 나누는 과정
  * 서브넷마스크
    * IP 주소에서 네트워크 부분과 호스트 부분을 구분

### 기타기술

* NAT(Network Address Translation)
  * 개념
    * 사설 IP 주소를 공인 IP 주소로 변환
  * 사용목적
    * 공인 IP 주소 절약
  * 종류
    * Static NAT
      * 1 : 1
    * Dynamic NAT
      * 1 : N
    * PAT(Port Address Translation)
      * N : M
* DNS(Domain Name System)
  * 개념
    * 도메인 이름과 IP 주소를 상호 변환
* Qos(Quality of Service)
  * 개념
    * 네트워크에서 서비스의 품질을 보장
* VPN(Virtual Private Network)
  * 개념
    * 공중 네트워크를 통해 사설 네트워크 연결
* DHCP(Dynamic Host Configuration Protocol)
  * 개념
    * 네트워크 장치에 IP 주소를 자동으로 할당

## 프로토콜

### 프로토콜

* 개념
  * 원활한 데이터교환을 위한 표준화된 통신 규약
  * 통신을 제어하기 위한 표준 규칙과 절차의 집합
* 기본요소
  * 구문
    * 데이터의 형식과 부호, 신호 레벨
  * 의미
    * 효율적이고 정확한 정보 전송 협조사항 및 오류 관리 제어 정보를 규정
  * 타이밍
    * 통신 속도 메시지 순서 제어
* 기능
  * 단편화 재결합
  * 캡슐화
  * 흐름제어
  * 오류제어
  * 혼잡제어
  * 동기화
  * 순서제어
  * 주소지정
  * 다중화
  * 경로제어

### 흐름제어와 오류제어

* 흐름제어
  * 수신측의 처리 능력에 맞춰 송신측에서 데이터의 전송량이나 속도 조절
  * Stop and Wait
    * 각 패킷 전송후 응답 받고 다음 패킷 전송
    * 오버 플로 방지 전송 효율 낮음
  * Sliding Window
    * 수신측에서 설정한 윈도우 크기 만큼 송신측에서 확인 응답 없이 패킷을 연속적으로 전송
    * 전송효율 크게 향상
  * Piggybacking
    * 수신측에 별도 ACK 프레임을 받지 않고 데이터 전송 프레임에 응답 기능을 포함
    * 양방향 통신에서 교차전송 하는 방식
* 오류제어
  * 전송중 발생하는 오류를 검출하고 정정
  * TCP 프로토콜 ARQ(Automated Repeat Request)
    * Stop and Wait ARQ
      * 데티 전송후 수신측으로부터 확인 응답을 받을때까지 대기 후 데이터 전송
    * Go Back N ARQ
      * 오류가 발생한 지점으로 돌아가 재전송
    * Selective Repeat ARQ
      * 오류가 발생한 패킷만 재전송
    * Adaptive ARQ
      * 오류율에 따라 재전송 방식을 변경
* 원인
  * 감쇠
  * 지연왜곡
  * 상호변조잡음
  * 충격잡음
* 전송오류 제어방식
  * 전진오류수정
    * 스스로 오류 검출
    * 해밍코드
    * 상승코드
  * 후진오류 수정
    * 오류발견시 송신측에 재전송 요구
    * 오류 제어는 ARQ 에 의해 수행
* 오류 검출
  * 패리티 검사
    * 데이터 블록끝에 패리티 비트 추가
  * 순환 중복 검사
    * 데이터 뒤에 오류 검출 코드인 FCS 추가
  * 체크섬
    * 간단한 방법으로 데이터 오류 검출
  * 해밍코드
    * 수신측에서 오류 검출 자동으로 수정
  * 상승코드
    * 순차적 디코딩 및 한계값 디코딩을 사용

## OSI 7계층

### OSI 7계층

* 개념
  * 네트워크 프로토콜을 기능별로 나눈 표준 모델
  * ISO(국제표준화기구) 에서 제정
  * 응용계층 -> 표현계층 -> 세션계층 -> 전송계층 -> 네트워크계층 -> 데이터링크계층 -> 물리계층
  * 트랜스포트 -> 세그먼트
  * 네트워크 -> 패킷, 라우터
  * 데이터링크 -> 프레임, 브릿지, 스위치
  * 물리 -> 비트, NIC, 리피터, 허브

### 계층별 특징

* 물리 계층(Physical Layer)
  * 데이터를 전기적 신호로 변환하여 통신 케이블 통해 전송
  * 비트(0,1) 전송
  * 단순 데이터전달, 에러 처리 않음
  * 통신 케이블, 랜카드, 리피터, 허브
* 데이터링크 계층(Data Link Layer)
  * 포인트 투 포인트 의 신뢰성있는 데이터 전송 보장
  * 물리 계층 오류 감지 수정
  * CRC(Cyclic Redundancy Check)사용
  * MAC 주소 지정
  * 스위치, 브릿지
* 네트워크 계층(Network Layer)
  * 데이터를 목적지까지 라우팅
  * 경로설정, 주소부여
  * 라우터, L3 스위치
* 전송 계층(Transport Layer)
  * 양 종단간 통신을 제어
  * 데이터 전송 신뢰성 보장
  * TCP, UDP
* 세션 계층(Session Layer)
  * 양 끝단의 응용 프로세스 간 통신 관리
  * TCP/IP 세션 생성 및 관리
* 표현계층(Presentation Layer)
  * 다른 데이터 표현 형식간의 변환을 담당
  * 데이터 암호화, MIME 인코딩
* 응용 계층(Application Layer)
  * 사용자 인터페이스, 응용 프로그램간 통신 담당
  * HTTP, FTP, SMTP, POP3, IMAP, Telnet

### 네트워크 장비

* LAN 카드
  * PC 와 네트워크 간 정보 교환
  * LAN 카드에는 MAC 주소가 할당 48비트 물리주소
* Hub
  * 네트워크 내 여러 노드를 연결
  * 네트워크에 연결된 모든 PC 들을 하나의 중앙지점에 모아줌
* Repeater
  * 디지털 신호를 증폭하여 신호가 약해지지 않고 먼 거리까지 전송될수 있도록 하는 장비
* Bridge
  * 두개 이상의 LAN 을 연결하여 하나의 확장된 네트워크 형성
* Switch Hub
  * 허브 기능에 스위치 기능 추가
  * 네트워크 트래픽 관리 및 효율적 전송을 위해 사용
* Router
  * 패킷을 목적지 까지 최적의 경로를 통해 전달
  * 네트워크 간 라우팅 및 트래픽 관리 담당
* Gateway
  * 서로 다른 네트워크 또는 프로토콜 간의 통신
  * 다른 유형의 네트워크 또는 프로토콜 연결

### 백본

* 백본 네트워크
  * 기간망으로 알려진 대규모 패킷 통신망
  * 빠르게 전송
* 백본 스위치
  * 네트워크 중심 모든 패킷이 지나감
  * 많은 트래픽 기가급 장비 사용
* 스위치 종류
  * L2 스위치
    * 데이터 링크 계층에서 운용 MAC 주소 기반 패킷 전송
  * L3 스위치
    * 네트워크 계층에서 운용 IP 주소 기반 패킷 전송
    * 라우팅 기능 탑재
  * L4 스위치
    * 전송 계층에서 운용 TCP/UDP 포트 기반 패킷 전송
    * 로드밸런싱 기능 탑재
  * L7 스위치
    * 응용 계층까지 운용 HTTP, FTP, SMTP 등 프로토콜 기반 패킷 전송
    * 주로 보안 장비에 사용
