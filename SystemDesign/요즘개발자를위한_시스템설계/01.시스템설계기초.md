## 시스템설계 기본
<hr/>

### 정의
<hr/>

#### 소프트웨어 시스템

* 특정 작업이나 일련의 작업을 수행하려고 함꼐 동작하는 컴포넌트나 모듈, 프로그램 집합
* 애플리케이션, 소프트웨어 개발시 고려 사항
  * 프로그래밍 언어
  * 에디터
  * 개발 방식(방법론)
  * 테스트 방식(TDD)
  * 유지보수 제공 여부

#### 분산 소프트웨어 시스템

* 여러 독립적인 컴포넌트나 프로세스 노드로 구성
* RPC, 메시지 전달, 발행/구독 같은 프로토콜을 통해 소통

#### 시스템 설계 과정

1. 요구사항 분석
2. 상위 수준 아키텍쳐 설계(컴포넌트, 모듈, 인터페이스)
3. 하위수준 상세 설계(시스템 내부 구조 동작 설계, 비즈니스 로직, 알고리즘)
4. 사용자 인터페이스 설계(API)
5. API 설계
6. 데이터베이스 설계

### 시스템 설계 유형
<hr/>

### 상위 시스템 설계

* 시스템 아키텍처 -  시스템의 전반적 구조
* 데이터 흐름 - 데이터가 수집되어 처리하는 과정
* 확장성 - 부하 처리 능력
* 장애 허용

#### 시스템 아키텍처

* 패턴
  * 모놀리식
  * 클라이언트 서버 - 클라이언트가 서버로 요청
  * 마이크로서비스
  * 이벤트 주도 - 비동기 이벤트 또는 메시지로 상호작용
* 고려 사항
  * 확장성
  * 유지보수성
  * 신뢰성 - 중단없이 안정적으로 동작할수 있는지
  * 지연시간

#### 데이터 흐름

* 고려사항
  * 데이터 수집 - 데이터가 어디서 오고 어떻게 가져올지
  * 데이터 저장
  * 데이터 처리 - 데이터 변환, 분석, 요약
  * 데이터 검색 - 응답시간, 부하 고려

#### 확장성

* 수직확장성 - CPU, 메모리등 추가
* 수평확장성 - 여러 컴포넌트, 인스턴스로 분할

#### 장애허용시스템

* 복제, 중복성,, 잠진적 성능 저하, 모니터링, 복구등을 신경써서 설계

### 하위 수준 설계

* 알고리즘
* 데이터 구조
* API
* 코드 최적화

#### 알고리즘

* 고려사항
  * 시간복잡도 - 데이터와 연산횟수 상관관계
  * 공간복잡도 - 데이터와 메모리 양간 관계
  * 트레이드 오프 - 시간복잡도와 공간복잡도의 균형

#### 데이터 구조

* 고려사항
  * 데이터 접근 방식 - 데이터 형태와 읽기, 쓰기 고려
  * 쿼리 성능 - 작업에 대한 시간 복잡도
  * 메모리 사용량

#### API

* 고려사항
  * 일관성 - 어떠한 모듈에서라도 일관된 설계
  * 유연성 - 기존 기능 영향 없어야함
  * 보안 - 인증, 권한, 유효성등 외부 접근과 데이터 유출 염두
  * 성능 - 지연시간 최소화
* `/api/v1/users -> /api/v2/users`

#### 코드 최적화

* 리팩토링
* 루프 언롤링 - 반복문의 횟수를 줄여서 성능 향상
* 메모리 제이션 - 이전 실행결과와 동일한 결과 다시 계산
* 병렬처리

### 시스템 설계 중요성

* 요구사항에 대한 명확한 이해
* 협업 시너지
* 설계 검토 및 피드백
* 높은 확장성
* 성능
* 유지보수성
* 비용 효율성
